Model
    Represents the data.
    Maintains data consistency.
    implements logic and rules - Business logic
    Publishes change events for Viewers to subscribe to.
    View
    Presents the data.
    multiple views - one view doing everything is a bad implementation
    • web pages
    • web services (API)
    UI - graphs, charts, diagrams. Possibly multiple views of the same data
    Subscribes to Model change events.
    Requests model changes.
    Requests model data.
    Controller
    Services requests from User.
    Services requests from View.
    Conditions and proxies requests to / from View and Model
